    Does the compiler compile code even if it warns you?

    It does compile even if you get an error. It does this because it thinks that you might have another file that you didn’t import yet that will solve the problem.

    What does the tsconfig file do? What can you change and what is the effect?

    The tsconfig configures how your compiler should behave. You can change it to make it not compile if there is a warning.

    Other options you can change include:

    • what format you want it to compile in

    • what version of the compiler you have(es3,es5,es6)

    • sourceMap true or false (gives access to typescript code in developer tools in the browser)

    • noImplictAny automatically sets the “any” type to anything that does not have a specific type. If you put it to true then it will give you a warning if you do not put a specific type on something.
